# UI 布局规范

## 整体布局

- **13×13 主画布**为视觉中心，在页面中**上下左右居中**
- 其他 UI 元素（楼层标识、右侧面板）基于画布位置进行**绝对定位**，不影响画布居中
- 画布自适应填满可用空间（连续缩放，非整数倍），支持 devicePixelRatio 高清渲染
- `imageSmoothingEnabled = false` 保持像素风格

## 楼层标识（Banner）

- 位置：画布**左上角上方**，绝对定位
- 格式：`F{层数} {层名}`，如 `F1 教学层`
- 样式：20px 加粗，颜色 `#ffd166`

## 右侧面板（Right Panel）

- 位置：画布**右侧**，顶部与画布**上对齐**，绝对定位
- 宽度：140px（900px 以下屏幕缩至 120px，700px 以下隐藏）
- 背景：`rgba(255,255,255,0.06)`，圆角 8px

### 面板结构

```
标题栏：{人物名字 加粗金色} {Lv.X 加粗金色}（exp/100）
─────────────────
ICON  HP         100

─────────────────

```

### 行布局规范（所有模块统一）

每行高度固定 **28px**，三列布局：

| 列 | 宽度 | 对齐 |
|---|---|---|
| Icon（sprite-icon） | 固定 24px | 左侧 |
| 文字标签（.label） | flex: 1（填充剩余） | 左对齐，左边距 4px |
| 数值（.val） | 最小 48px | 右对齐，白色加粗 |

### 分隔线

- 使用 `<hr class="panel-divider">`
- 样式：1px solid `rgba(255,255,255,0.15)`，上下 margin 4px
- **道具** 和 **怪物** 标题的上方和下方各一条分隔线

### 标题行

- 颜色：`#ffd166`，加粗
- 高度：28px（与数据行一致）
- 无额外 margin

### 怪物列表

- 仅显示本层存在的怪物种类，有几种显示几条
- **不显示**战斗伤害预估
- 数值格式：`HP/攻/防`（如 `25/12/2`）

## 战斗确认

- **无损失**（damageTaken == 0）：直接进入战斗，不弹窗
- **有损失**（damageTaken > 0）：弹出模态弹窗，玩家确认后才战斗
  - 弹窗标题："战斗确认"
  - 弹窗内容：对手名称及属性（HP/攻/防）、HP 变化（当前 → 战后）
  - 如果会被击败，额外显示红色警告 "⚠ 你会被击败！"
  - 按钮："战斗"（金色）/ "取消"（灰色）
  - 弹窗期间屏蔽所有移动输入
- 样式：深蓝底（`#1a1a2e`）+ 金色边框（`#ffd166`），半透明黑色遮罩
- 死亡后显示 "你被击败了..."，2 秒后自动重置游戏

## 消息气泡

- 位置：页面顶部居中，绝对定位
- 样式：圆角胶囊，半透明黑色背景
- 显示时长：1.8 秒后自动隐藏

## 重新开始按钮

- 位置：画布右上角上方，绝对定位
- 样式：12px 灰色文字，细边框圆角按钮，hover 变亮
- 点击后清除存档并重新开始游戏

## 死亡弹窗

- 玩家 HP 归零时弹出模态弹窗
- 标题："你被击败了"
- 内容：显示被哪个怪物击败
- 按钮：仅"重新开始"（清除存档，重新开始）
- 样式：同战斗确认弹窗（深蓝底 + 金色边框 + 遮罩）

## 进度保存

- 存储位置：`localStorage`，key 为 `birthday-tower-save`
- 保存时机：战斗结束、拾取物品、开门、换层、商店购买
- 保存内容：当前楼层、玩家属性（HP/攻/防/金/经验/等级/钥匙/位置/朝向）、所有楼层的 grid 状态
- 页面刷新时自动恢复进度
- "重新开始"按钮和死亡时清除存档

## 移动端适配

- 700px 以下隐藏右侧面板
- 触屏设备显示方向键 D-pad（右下角，190px）
- 禁用双击缩放、长按菜单、文字选择
